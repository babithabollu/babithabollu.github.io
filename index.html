About Me

The engineer behind the models

Open to roles

I didn't start in AI — I started in the engine room. Five years managing Linux infrastructure for Dell taught me something most ML engineers never learn: production systems are unforgiving. Every design decision has a blast radius.

That foundation shaped how I build AI. When I moved into machine learning and eventually into LLMs and agentic systems, I brought that instinct for reliability with me. At Fifth Third Bank, I architected multi-agent pipelines that extract structured KPIs from thousands of financial PDFs — not as a demo, but in production, at scale, with 99.9% uptime and automated drift detection.

What I love is the full stack of this problem: choosing the right chunking strategy for a RAG system, tuning embedding models, designing the FastAPI layer that can handle concurrent AI query loads without flinching — and then wiring it all together with CI/CD so the next deploy is boring. Boring deploys mean the system works.

I specialize in the messy middle between "AI research" and "product that ships." If you're building something real with LLMs, multi-agent systems, or ML pipelines — let's talk.

Case Studies

Deep dives into the hard problems

The Problem

Analysts at a major bank were manually reviewing thousands of unstructured financial PDFs to extract KPIs — a process that took days, introduced human error, and couldn't scale. The question wasn't whether to automate it, but how to do it reliably in a regulated environment where wrong answers have real consequences.

The Solution

Built a multi-agent LangGraph workflow: one agent handles document parsing via Azure Document Intelligence, another validates extracted KPIs against business rules, a third routes exceptions to human review. Backed by hybrid RAG search and deployed on Kubernetes with MLflow monitoring and automated retraining.

Drift Detection

The Problem

Enterprise teams wanted LLMs to answer questions about internal systems, but every team had a different database schema and documentation format. Custom integrations for each were expensive, slow, and created security risks when sensitive data was exposed ad-hoc.

The Solution

Implemented Model Context Protocol (MCP) servers that standardize how internal databases and documentation are exposed to LLM clients. One protocol, any client — with access controls baked in. Combined with LangChain semantic search over Pinecone for retrieval accuracy.
